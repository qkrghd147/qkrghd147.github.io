## 자료형

**자료(의)형(태)**이라는 문법이 왜 존재하는지에 대해 설명할 수 있다.

**내 말을 잘 알아듣는 컴퓨터 만들기**

우리와 비슷하게 생각하는, 자료형 없는 컴퓨터 => 컴퓨터가 직접 생각해야 할것이 너무 많음... => 너무 느려짐 => 미리 자료형을 사람이 알려줌

초기 자료형은 문자와 숫자로 구분되었다

but python은 자료의 **종류 구분** + 자료의 관리를 위한 **자료형** 까지 갖춤

웹사이트 개발을 위한 자료의 관리를 위한 자료형은 리스트와 딕셔너리를 배워봄(튜플, 집합도 있음)

## 관리를 위한 자료형

리스트(튜플), 딕셔너리에 대해 알 수 있다.

해당 자료형이 등장한 이유, 어떤 상황에서 어떤 방법으로 어떤 데이터들을 관리할 것인가

**리스트 자료형**

ex) 나의 성적 = '90', '86', '98'

리스트 : **변할 수도 있는** 데이터들을 **나란히** 묶어주는 자료형

**튜플 자료형**

튜플 : **변할 수 없는(변하면 안되는)** 데이터들을 나란히 묶어주는 자료형

**딕셔너리(해쉬) 자료형**4

대응이 되는 표현해주는 자료형

우리는 **KEY**를 통해서 **VALUE**를 찾는다

dictionary = {key1:value1, key2:value2,....}

key는 중복되서도, 변해서도 안된다.

ex) 운동선수 = {"김연아":"피겨스케이팅", "박지성":"축구",....}, 운동선수['김연아']=='피겨스케이팅'

## 문자열 내장함수

* len(문자열 변수) = 문자열의 길이를 알려줌 (인덱싱과 달리 1부터 생각)
* 문자열 변수.count('특정 문자') = 문자열 변수에서 '특정 문자'가 몇번 나왔는지 알려줌
* 문자열 변수.split() = 공백을 기준으로 문자열을 나누어 리스트화 시킴, split('특정문자')일경우 특정문자를 기준으로 문자열을 나눔!
* 문자열 변수.find('문자'), 문자열 변수.index('문자) = 특정 문자 인덱스 찾기

## 리스트 내장함수

* len(변수) = 리스트의 길이를 알려줌
* 변수.sort() = 리스트의 원소를 오름차순으로 정렬
* 변수.index() = 리스트 내의 특정 원소 익덱스를 반환해주는 함수
* 변수.count() = 리스트 내의 특정 원소의 개수를 알려주는 함수

## 딕셔너리 내장함수

* 딕셔너리 변수안에 키:밸류 **추가하기** : 딕셔너리 변수['추가 키']='추가 밸류'
* 딕셔너리 변수안에 키:밸류 **삭제하기** : del 딕셔너리 변수['키'] => 딕셔너리 말고도 튜플과 리스트에도 사용된다.
* KEY로 VALUE얻기 : 딕셔너리 변수.get('키') => 장고에서도 많이 사용됨. 중요!

## 제어문(1) - 분기문

제어문 - 제어해주는 문법

무엇을 제어하느냐? 코드의 흐름을 제어한다

분기문(=if문) : 컴퓨터에게 선택의 여지와 조건을 부여
```
if(조건): 조건이 참일 때 실행할 코드
```
* 등호 두개(==) : 값이 일치하는지 알아보는 연산자
* 등호 한개(=) : 대입을 하는 연산자

else+if = elif

## 제어문(2) - 반복문

반복문 : for, while이 있음
```
for 반복제어변수 in 반복대상 : 반복 실행할 코드
```
* 반복 대상 = 반복하고싶은것(리스트, 튜플, 문자열 .....)
* 반복제어변수 = 반복 대상을 하나하나 가져오며 명령을 수행

**range 함수**

* range(x,y) => x이상 y미만의 수 리스트로 반환
* range(x) => 0부터 x미만의 수 리스트로 반환

for문 : 이 반복대상을 반복하세요!

while문 : 이 조건을 만족하는 동안 반복하세요!

**while(조건) : 반복 실행할 코드**

조건이 항상 참이면 = 무한 루프를 형성

반복문을 억지로 탈출시키는 명령어 = break => 무한루프랑 같이 쓴다

일단 반복하다가 if(조건)에 해당하면 break해!

## 함수

프로그래밍에서의 함수란? 

코드를 기능으로 묶은 단위, 코드의 기능 단위

```
def 함수이름(인자) : 실행코드와 리턴값
```

**왜 필요할까??**

ex) SNS에는 스토리 기능, 사진기능, 동영상기능, 댓글기능.....

하나의 코드로 작성시 에러 발생하면 찾기 힘듦 ㅠ => 기능별로 코드를 묶어서 코드를 작성

엄청 긴 코드를 기능별로 나눠 최적화 => Divide & Conquer

함수의 입구와 출구 : 원인(인자)이 박스에 드감, 결과(리턴)가 나옴

* 인자 : 개수는 내맘대로!
```
def function(x,y)...... => 호출시 : function(num1,num2)
```
* 리턴값 : 함수가 제 기능을 다 한 결과, 리턴값은 항상 1개(기능의 결과로 하나만 반환해줘야 함!)

리턴값이 없을수도있다

함수의 안과 밖 : 변수의 범위

함수의 안 : 함수만의 세상 => 밖과는 아예 독립적!! 

* 지역변수 : 함수 안에서만 영향력을 갖는 변수

지역 변수를 바깥으로 꺼내쓰고 싶어! => 가능

* 전역변수 : 코드 전체의 영역에서 영향력을 갖는 변수 => '**global** 변수'와 같이 선언

but 지역변수>>전역변수 => 함수의 본질에 기인 = 코드를 기능별로 정리정돈! = 함수를 통해서 영역구분을 하겠다.

수많은 전역변수가 있을경우 함수의 존재자체가 의미가 없어진다.
